# AI Dev Prompts — Hobbeast v1.5.5

## Objective
Stabilize the admin bulk selection flow and the event create/edit flow without regressions.

## Required implementation focus

- Treat no-filter bulk preview as local select-all over loaded profiles.
- Map backend preview IDs to UI row selection through both `profiles.id` and `profiles.user_id`.
- Filter null IDs out of preview responses.
- Keep bulk apply compatible with profile IDs and user IDs.
- Mark required event fields visually.
- Disable event submit/save until mandatory fields are present.
- Write `start_time` compatibility field alongside `event_date` / `event_time`.
- Guarantee non-null `place_categories` on event writes.
- Add database compatibility migration for start_time/place_categories/organizer compatibility.

## Validation checklist

- [ ] No-filter bulk select checks all loaded profiles.
- [ ] Preview count and checked rows are equal.
- [ ] Single-row deselect only affects that row.
- [ ] Create Event button stays disabled until required fields are filled.
- [ ] Event create writes `start_time`.
- [ ] Event edit writes `start_time`.
- [ ] Build passes.
- [ ] Changelog and lessons are appended.
