# AI Dev Prompts — v1.5.4

## Objective
Fix the remaining admin bulk-selection mismatch and related admin reliability defects without introducing regressions.

## Mandatory reading order
1. AI Developer Prompting Master Manifesto
2. controller / governance entrypoint
3. design-master-rules
4. changelog.md
5. codingLessonsLearnt.md
6. latest relevant versioning artifacts
7. current code and current error logs

## Required implementation instructions
- Treat the currently rendered admin table row as the canonical UI entity.
- Keep bulk preview and row checkbox state on the same row identity.
- Support historical profile rows without assuming a guaranteed auth-linked `user_id`.
- Keep batch apply compatible with both profile-level and auth-level resolution.
- Remove client reliance on fragile `upsert(... onConflict ...)` where the actual DB constraint cannot be guaranteed.
- Extend function config coverage so admin invoke tooling does not remain blocked by avoidable JWT verification drift.
- Preserve append-only documentation and update changelog / lessons.

## Concrete patch goals
### AdminUsers.tsx
- Canonical UI selection state must stay on `profile.id`.
- Preview response must be mapped to selected rows immediately.
- Support fallback mapping from `selectedUserIds` to visible profile rows when needed.
- Keep accurate selected count.
- Single-row deselect must only affect one row.

### admin-bulk-user-actions edge function
- Preview must return `selectedProfileIds` as the canonical row key.
- It may also return `selectedUserIds` as secondary data.
- Apply must accept `profileIds` and also tolerate `userIds` for backward compatibility.
- Delete / activate / deactivate must not assume every selected record has a valid `user_id`.

### Catalog / preferences
- Replace `upsert(... onConflict: 'slug')` and `upsert(... onConflict: 'user_id')` with select-then-update/insert flows.

### Edge function config
- Ensure `sync-local-places` and `place-search` are declared in `supabase/config.toml` with `verify_jwt = false`.

### Accessibility
- Add title/description coverage to recurring dialogs that currently trigger warning noise.

## Validation steps

- Test real-user filter preview.
- Confirm UI checkboxes highlight rows after preview.
- Confirm deselecting one row does not reset all.
- Confirm batch action payload still works.
- Confirm hobby catalog sync no longer throws 42P10.
- Confirm production build succeeds.
