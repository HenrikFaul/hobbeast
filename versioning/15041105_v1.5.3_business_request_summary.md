# Hobbeast v1.5.3 Business Request Summary

## Scope
Log-driven stabilization round based on uploaded repo snapshot, event creation errors, and multiple browser logs.

## Business goal
Stabilize the admin and event flows without regressions, preserve working batch user generation, and eliminate repeated runtime noise that blocks core admin operations.

## Requested outcomes
- Keep mass user generation working.
- Fix event creation when no provider-derived place categories are present.
- Remove repeated 401 edge-function failures for `sync-local-places` and `place-search`.
- Fix catalog sync failures caused by missing `ON CONFLICT`-compatible uniqueness.
- Reduce notification preferences persistence fragility.
- Remove dialog accessibility warnings.
- Append changelog and coding lessons.

## Root causes confirmed from logs
1. `sync-local-places` and `place-search` were called from the frontend but the function gateway returned 401.
2. `hobby_categories?on_conflict=slug` failed with `42P10`, meaning the expected unique/exclusion constraint was not present.
3. Event creation failed because `events.place_categories` received `null` while the database required a non-null value.
4. Several dialogs were rendered without description metadata, producing accessibility warnings.
5. The repo still mixed fragile upsert patterns with schema assumptions that were not guaranteed in the current database.

## Implemented fix strategy
- Edge-function config hardening in `supabase/config.toml`.
- New SQL migration for log-driven integrity fixes.
- Event create/edit payload fixes.
- Catalog sync changed from fragile upsert-on-slug to select/update-or-insert flow.
- Notification preferences changed from fragile upsert-on-user_id to select/update-or-insert flow.
- Dialog accessibility descriptions added.
