# Business Request Summary — v1.5.6 / 15041108

## Scope
- Fix admin bulk user selection so preview count, UI badge, and applied actions stay aligned.
- Fix event create/edit so required fields are visibly marked, submit is disabled until valid, and compatibility `start_time` is always sent.
- Remove remaining fragile `onConflict` assumptions in catalog and notification preference writes.
- Reduce repeated dialog accessibility warnings.

## Business expectation
- A "Mindegy / Mindegy / üres" bulk filter combination must effectively select all matching users and show the same count in backend response and UI.
- Admin bulk actions must not fail because the frontend and edge function disagree on `userIds` vs `profileIds`.
- Event creation must not allow avoidable NOT NULL database failures.
- The interface must clearly communicate required fields before submission.

## Non-goals
- No broad redesign of the admin information architecture.
- No schema rewrite beyond minimal compatibility migration.
