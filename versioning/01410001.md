# Hobbeast redesign package — 01410001

## Cél
A Hobbeast teljes designrendszerének újratervezése úgy, hogy:
- semmilyen már működő funkció ne sérüljön,
- a redesign ne írja át a route-okat, auth flow-kat, Supabase szerződéseket vagy backend logikát,
- a jelenlegi képernyők egységes, modern, mobil-first, jobb hierarchiájú és jobban skálázódó vizuális rendszert kapjanak.

## Miért kell redesign?
A repo audit alapján a vizuális rendszer jelenleg részben már erős, de még széttartó:
- van központi tokenkészlet az `src/index.css` fájlban, de nincs teljes értékű page shell és layout grammar;
- több route hasonló feladatot más-más hero / card / filter / spacing logikával old meg;
- a dashboard jellegű felületek (`OrganizerDashboard`, `Admin`, `Profile`) más vizuális nyelven beszélnek, mint a discovery / landing / event oldalak;
- a repo még tartalmaz default Vite maradványt (`src/App.css`), ami design governance szempontból takarítandó;
- a fő navigáció nem tükrözi eléggé az app valódi információ-architektúráját;
- a footer és az oldalfejlécek nem alkotnak egységes, brand-szintű rendszert;
- több képernyőn jól működő funkciók vannak, de a vizuális hierarchia, grouping és CTA prioritás gyenge vagy következetlen.

## Kötelező keretek
A redesign során kötelező:
- a működő üzleti logikák, route-ok, auth, organizer, admin, Supabase és edge function szerződések megtartása;
- a regresszió minimalizálása;
- a már működő create/edit/join/leave/profile/admin flow-k megőrzése;
- a meglévő komponensökoszisztéma (React + Vite + Tailwind + shadcn/ui + framer-motion) használata;
- a jelenlegi brand (Hobbeast, meleg narancs + zöldes akcentus, közösségi / élmény fókusz) megtartása, de kiforrottabb, prémiumabb rendszerbe emelve.

## Feltárt fő UI problémák

### 1. Nincs közös page shell
A legtöbb route egyedi `pt-24 pb-16 min-h-screen` kezdéssel és saját hero/logikai mintával él. Emiatt:
- nincs egységes oldalritmus,
- nincs shared header rendszer,
- nincs közös page-intro grammar,
- a discovery / events / about / admin nézetek szétesnek.

### 2. Navigáció túl szűk és lapos
A `Navbar` jelenleg:
- nem mutatja a `Felfedezés` route-ot,
- kevéssé hangsúlyozza a Hobbeast brandet,
- desktopon egyszerű linklistaként működik,
- mobile-on funkcionális, de nem prémium élmény.

### 3. Hero és page-intro rendszerek inkonzisztensek
- `Index` külön hero komponensekből áll;
- `About` külön vizuális nyelvet használ;
- `Admin` egy szinte teljesen más belépő layoutot használ;
- a dashboard és marketing jellegű oldalak közötti átmenet durva.

### 4. Tokenek jók, de nem eléggé kiforrottak
Az `index.css` használható, de:
- a shadow rendszer visszafogott és helyenként lapos;
- a gradient rendszer jó alap, de nincs magasabb szintű surface grammar;
- hiányzik a page-szintű utility rendszer (`surface-panel`, `metric-tile`, `page-stack`, stb.).

### 5. Még bent maradt Vite maradvány
A jelenlegi `src/App.css` teljesen idegen a projekttől, design governance alapján cserélendő.

## Két redesign út rövid összehasonlítása

### Opció A: konzervatív shared-shell redesign
- csak globális layout, navbar, footer, header, spacing és token finomítás;
- kisebb regressziós kockázat;
- gyorsabb delivery;
- de a nagy route-ok szerkezeti problémáit kevésbé oldja meg.

### Opció B: route-szintű teljes redesign – tisztább UX, de nagyobb regressziós kockázat
- minden fő route új page shellt, új hierarchy-t, új layout csoportosítást kap;
- erősebb, egységesebb termékélmény;
- nagyobb módosítási felület.

### Választás: kontrollált teljes redesign, ahol a funkciók és szerződések maradnak, de a layout, hierarchia és komponensrendszer egységesen újul meg.

## Javasolt redesign stratégia

### 1. Új shared design grammar
Bevezetendő közös minták:
- `PageHeader` / `SectionHeader`
- egységes page shell wrapper
- `surface-panel` utility
- `metric-tile` utility
- egységes hero-chip / eyebrow rendszer
- egységes empty-state / filter-toolbar / dashboard-intro minták

### 2. Új globális layout karakter
- lágyabb, prémiumabb felületek
- több rétegzett blur/surface használat
- egységes shadow grammar
- modernebb, fókuszáltabb CTA rendszer
- jobb content-width kontroll
- erősebb responsive töréspontok

### 3. Új navigációs és footer rendszer
- a valós route-struktúrához igazított navbar
- `Felfedezés` visszahozása első szintre
- erősebb organizer affordance
- brand + context + CTA jobb aránya
- strukturált footer valódi platform-információval

### 4. Route-onként redesign
A fő route-ok külön újratervezendők:
- `/` → marketing + entry dashboard
- `/explore` → discovery-first taxonómia
- `/events` → eseményböngésző, szűrő- és kártyahierarchia
- `/events/:id` → event detail storytelling + action clarity
- `/about` → brand/mission/facts oldal
- `/auth` → konverzió-orientált belépési élmény
- `/profile` → dashboard jellegű személyes központ
- `/organizer` → multi-panel operations dashboard
- `/admin` → operation shell a jelenlegi tabok megtartásával

## Route redesign részletek

### `src/pages/Index.tsx`
Új irány:
- hero ne csak marketing legyen, hanem entry-point dashboard is;
- legyen benne gyors útvonal a `Felfedezés`, `Események`, `Profil / Közösség` irányokra;
- a meglévő `FeaturesSection`, `ResearchSection`, `CTASection` maradhat, de új shellbe rendezve;
- a top hero kapjon kétoszlopos premium layoutot: bal oldalt value prop + CTA + stats, jobb oldalt image / community visual panel.

### `src/pages/About.tsx`
Új irány:
- ne egyszerű marketing szekciók sora legyen, hanem brand story page;
- page header + mission blocks + problem/solution + research stats egy közös vizuális rendszerben;
- a jelenlegi jó tartalom maradhat, de modernebb groupinget kapjon.

### `src/pages/Admin.tsx`
Új irány:
- admin intro kapjon page header shellt;
- a tabbar kapjon erősebb operation dashboard stílust;
- a jelenlegi tab tartalom marad, csak a shell és a belépő ritmus változik.

### `src/pages/Explore.tsx`
Új irány:
- discovery experience legyen, ne csak grid switching;
- page header + search strip + left/right hierarchy vagy erősebb breadcrumb/density rendszer;
- kategória / alkategória / activity nézetek kapjanak közös card grammar-t;
- a stat summary és az activity profile summary modern info-panellé váljon.

### `src/pages/Events.tsx`
Új irány:
- page header + quick actions + source filters + discovery toolbar;
- a jelenlegi nagyon sűrű filterrész szétszedendő logikai blokkokra;
- a category modal maradhat, de a nyitógomb és a kiválasztott state-ek újragondolandók;
- az event cardok vizuálisan hangsúlyosabbak, jobban pásztázhatók legyenek;
- a távolságszűrő külön surface-panelt kapjon.

### `src/pages/EventDetail.tsx`
Új irány:
- storytelling-szerű event hero;
- action hierarchy tisztítása: own / joined / external / sample state-ek jobban elváljanak;
- info cards, venue details, description, trip plan és CTA-k egységes történetben jelenjenek meg;
- a current action buttons kapjanak tisztább priority rendszert.

### `src/pages/Auth.tsx`
Új irány:
- a jelenlegi layout alapvetően jó, de finomítható:
  - jobb vizuális balance,
  - konzisztensebb brand storytelling,
  - mobile-on fókuszáltabb spacing,
  - shared design grammarhoz igazítás.

### `src/pages/Profile.tsx`
Új irány:
- személyes dashboardként kezelendő;
- page header + bal oldali fő szerkesztési blokkok + jobb oldali utility kártyák;
- section hierarchy újragondolása;
- hobby / location / personal blocks egységesebb shellt kapjanak.

### `src/pages/OrganizerDashboard.tsx`
Új irány:
- operations dashboard shell;
- intro header + event summary metrics + tab shell egységesítése;
- attendees / messages / analytics sokkal érettebb admin/ops karaktert kapjanak;
- a participant detail sheet maradhat, de összhangba hozandó az új rendszertonnal.
