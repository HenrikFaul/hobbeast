# Hobbeast v1.5.5 Business Request Summary

ID: 15041107  
Version: v1.5.5  
Date: 2026-04-11

## Request scope

1. Fix the remaining bulk-selection mismatch so the UI selection always matches the backend preview.
2. When no effective bulk filters are set (`Mindegy`, `Mindegy`, empty day fields), selecting should mark all loaded profiles.
3. Fix event creation so required datetime/location data is enforced before submission.
4. Prevent event create from failing due to missing compatibility datetime fields such as `start_time`.
5. Preserve regression safety and append the new lessons and changelog entries.

## User-visible problems observed

- Bulk preview response reported far more matches than the UI actually checked.
- Checkbox highlighting stayed partially empty even when the backend preview returned many IDs.
- Event creation could be submitted with incomplete required fields.
- Event creation failed with database/runtime complaints around missing required compatibility fields.

## Business expectation

- Preview count and actual checked rows must be consistent.
- Admin must be able to trust “select all by filters”.
- Required event fields must be clearly marked.
- The create button must stay disabled until required fields are valid.
- Event writes must remain compatible with the current mixed-schema environment.

## Guardrails

- No regression on existing bulk actions.
- No regression on existing event create/edit UX beyond stricter validation.
- Keep append-only documentation and versioning.
