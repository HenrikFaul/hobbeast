# 14040713_v1.4.7 AI dev prompts - Hobbeast local sync admin UX and scheduler

## Business request
The Hobbeast local address-table backend now works, but the admin UI still cannot reliably import data because it waits synchronously for long-running batch syncs. The user requested ready-to-replace patch files so they do not have to hand-edit risky code regions.

## Required outcome
1. Make the admin UI stable for local place batch imports.
2. Add automatic scheduled batch execution every configurable number of minutes.
3. Allow the admin UI to configure:
   - interval_minutes
   - task_batch_size
   - provider_concurrency
   - radius_meters
   - geo_limit
   - tomtom_limit
4. Ensure the edge function reads those runtime values from `app_runtime_config`.
5. Keep the solution append-friendly and suitable for future governance-based tracking.

## Delivered files
- `src/components/admin/AdminEventbrite.tsx`
- `supabase/functions/sync-local-places/index.ts`
- `supabase/migrations/20260407223000_local_places_scheduler_and_enqueue.sql`
- changelog / lessons append snippets

## Implementation model
- UI starts a batch by RPC enqueue instead of directly waiting for the full sync.
- UI polls `sync-local-places` status every few seconds.
- Scheduler can launch periodic batches without code changes.
- Batch/provider limits are fully runtime-configurable from the admin UI.
