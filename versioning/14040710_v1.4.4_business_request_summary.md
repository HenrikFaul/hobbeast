# hobbeast v1.4.4 - business request summary

## Kérés összefoglaló
- A venue finder local-first módon működjön magyarországi országos venue/POI katalógussal.
- A katalógus Geoapify + TomTom batch syncből épüljön, és naponta frissíthető/ütemezhető legyen.
- A keresés elsődlegesen lokális katalógusból adjon találatot, majd provider fallbackkel egészítse ki.
- Az admin felületen maradjon az újratöltési lehetőség, és a common admin inventory maradjon rendezett, kategóriás.

## Leszállított implementációs delta
- `sync-local-places` edge function országos HU tile alapú bejárást kapott, deduplikálással és részleges-hiba visszajelzéssel.
- `place-search` edge function local-first egyesített keresést használ (`local + geoapify + tomtom`), a lokális találatok pontozási prioritással jelennek meg.
- Készült napi ütemezéshez SQL helper migráció (`schedule_daily_local_places_sync`, `unschedule_daily_local_places_sync`).
- A common admin integráció-inventory gépi kategória mezőt használ, ebből történik a stabil UI csoportosítás.

## Scope
- Shared/global: place search edge orchestration, local catalog sync, common admin integration inventory rendering, versioning/changelog/lessons append.

## Nem-funkcionális cél
- Legkisebb regressziós kockázat: meglévő import/provider admin flowk érintetlenek maradnak.
