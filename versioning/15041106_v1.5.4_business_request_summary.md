# Business Request Summary — v1.5.4

**Versioning ID:** 15041106  
**Version:** v1.5.4  
**Date:** 2026-04-11

## Request context
The current Hobbeast admin UI still shows several production-level defects after the previous bulk-user and admin fixes. The latest screenshots and logs show that:

- bulk selection preview returns IDs but rows are not highlighted in the UI,
- real-user bulk selection can return many `null` values,
- repeated `401 Unauthorized` errors still appear for `sync-local-places` and `place-search`,
- repeated `42P10` errors still appear for hobby catalog sync because the client relies on `on_conflict=slug`,
- dialog accessibility warnings are still present.

## Business goal
Stabilize the admin workflows so that:

1. bulk preview selection and UI highlighting behave consistently,
2. bulk actions remain usable even when some historical profiles do not have a reliable `user_id`,
3. common admin tooling no longer produces avoidable 401 noise for configured edge functions,
4. hobby catalog sync and preference save no longer depend on fragile `ON CONFLICT` assumptions,
5. accessibility warnings are reduced in recurring admin dialogs.

## Expected behavior
- Bulk selection preview must return row identifiers that the table can immediately highlight.
- Selection count must match the rows that visually show as selected.
- Deselecting one row must not clear unrelated rows.
- Batch apply actions must accept the same identifiers that the preview/UI are using.
- Rows without auth-linked `user_id` must still be handled safely where possible.
- `sync-local-places` and `place-search` must be configured for the intended admin invoke behavior.
- Hobby catalog sync must complete without `42P10` upsert failures.
- Notification preference save must not rely on missing uniqueness constraints.
- Dialogs should include title/description coverage for accessibility tooling.

## Scope
### In scope
- `src/components/admin/AdminUsers.tsx`
- `supabase/functions/admin-bulk-user-actions/index.ts`
- `src/components/admin/AdminCatalog.tsx`
- `src/components/NotificationPreferencesCard.tsx`
- `src/components/ui/command.tsx`
- `supabase/config.toml`
- append-only updates to `changelog.md` and `codingLessonsLearnt.md`

### Out of scope
- broader event lifecycle redesign,
- deep schema cleanup across all legacy tables,
- unrelated visual redesign.

## Regression risks to control
- bulk apply payload mismatch between frontend and edge function,
- deleting or deactivating the wrong records because of identifier drift,
- breaking catalog sync while trying to fix slug conflicts,
- introducing new accessibility regressions in dialog wrappers,
- assuming every profile has an auth-linked `user_id`.

## Validation checklist
- Bulk preview on generated users selects and highlights rows correctly.
- Bulk preview on real users no longer results in a visually empty selection when rows match.
- Selected counter matches highlighted rows.
- Single-row deselect only affects the chosen row.
- Catalog sync no longer emits `42P10` for `hobby_categories` slug upsert.
- Notification preference save still works.
- `npm run build` succeeds.
